# 場景切換與語音動畫移植說明

本文件說明了從「人間版 V3」移植到 `QSanguosha-v2` 的場景背景切換 (`changeBackground`) 與語句動畫特效 (`setLoopEmotion`) 功能。

## 功能簡介

這兩個 API 允許開發者透過 Lua 腳本或後端 C++ 直接觸發客戶端介面的視覺變化。

### 1. 更改場景背景 (`changeBackground`)
此功能允許在遊戲進行中動態替換桌面的背景圖。

*   **Lua 調用法**:
    ```lua
    room:changeBackground("background_name")
    ```
*   **參數說明**:
    *   `background_name`: 背景檔案的名稱。程式會自動尋找路徑 `image/system/backdrop/<background_name>.jpg`。
*   **運作原理**: 後端 `Room::changeBackground` 會發送 `S_ANIMATE_LIGHTBOX` 指令，參數帶有 `background=` 前綴。前端 `RoomScene::doLightboxAnimation` 解析到該前綴後，會加載對應的圖片並更新 `m_tableBg`。

### 2. 設定循環/語句動畫 (`setLoopEmotion`)
此功能用於在玩家頭像或介面上顯示特定的循環動畫或是提示文本（通常用於對應特定的語音或情緒）。

*   **Lua 調用法**:
    ```lua
    room:setLoopEmotion(player, "emotion_name")
    ```
*   **參數說明**:
    *   `player`: 目標玩家物件 (`ServerPlayer`)。
    *   `emotion_name`: 特效或語句的關鍵字。
*   **運作原理**: 調用 `doAnimate(S_ANIMATE_LIGHTBOX, "loopmove=" + emotion_name, player->objectName())`。客戶端會根據 `loopmove=` 參數執行相應的文字動畫顯示。

## 核心修改清單

### 後端實作 (C++)
*   **`src/server/room.h`**: 於 `Room` 類別中新增了函數宣告。
*   **`src/server/room.cpp`**: 實作了 `setLoopEmotion` 與 `changeBackground`，並透過 `doAnimate` 發送網路封包至客戶端。

### Lua 綁定 (SWIG)
*   **`swig/sanguosha.i`**: 將上述兩個函數暴露給 Lua 層，確保 Lua 腳本中的 `sgs.Room` 物件可以正常呼叫這些方法。

### 前端處理 (UI)
*   **`src/ui/roomscene.cpp`**: 確保 `doLightboxAnimation` 能正確識別 `background=` 指令。
    *   *註：針對 `loopmove=`，若無特定 QML 動畫定義，預設將以文字形式顯示。*

## 注意事項
1.  **資源檔案**: 調用 `changeBackground` 前，請確保 `image/system/backdrop/` 下存在對應的 `.jpg` 檔案。
2.  **編譯需求**: 由於修改了 SWIG 接口檔案 (`.i`)，修改後必須重新編譯整個專案，以產生最新的 Lua Wrapper 程式碼。

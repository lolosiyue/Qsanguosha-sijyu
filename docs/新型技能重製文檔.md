# 新型技能重製文檔

## 概述

本專案將傳統技能重構為兩大類新式技能：

| 類別 | 基類 | 特性 | 運作方式 |
|------|------|------|---------|
| **主動觸發技能 (Active V2Skill)** | `TriggerV2Skill` | 受事件觸發，有 cost/pay/effect 流程 | `RoomThread::triggerV2Skills` 驅動 |
| **被動全域修正技能 (Passive Correct Skill)** | `DistanceSkill` / `MaxCardsSkill` / `TargetModSkill` / `AttackRangeSkill` | 無須觸發，引擎持續計算 | `Engine::correctXxx()` 在每次查詢時自動計算 |

兩者共享同一套**多實例基礎設施**（同一玩家可持有多個同名技能實例），以及 **modified_amount 後置數值覆寫系統**。

---

## 第一部分：多實例基礎設施

### 實例 ID 機制

每個 Skill 實例由 `instanceId` 區分，儲存於 `Player::acquired_skills`（`QStringList`）。

| 儲存格式 | instanceId | 說明 |
|---------|-----------|------|
| `"skillName"` | 0 | innate（武將天生）或無指定 |
| `"skillName#1"` | 1 | 第一個 acquire 的實例 |
| `"skillName#2"` | 2 | 第二個 acquire 的實例 |

### Player 層 API

```cpp
// 取得該技能所有 instanceId（含 innate if found）
QList<int> getSkillInstanceIds(const QString &skill_name) const;

// 過濾 isSkillInvalid 後的「有效」實例 ID
QList<int> getValidSkillInstanceIds(const QString &skill_name) const;
```

### 實例計數規則

對每個技能名稱：
```
有效實例數 = innate（instanceId=0，若在 skills 清單中且有效）
             + acquired #1..#N 逐一過濾 isSkillInvalid 後存活者
```

### acquireSkill 行為

```cpp
// 自動分配新 instanceId（max+1）
acquireSkill(player, "baGua");

// 精確指定
acquireSkill(player, "baGua#3");
```

### 技能無效化（SkillInvalidity）

支援**逐實例失效**，Record 格式：`"skillName#instanceId|sourceName|reason"`。

| Record | 效果 |
|--------|------|
| `"all\|src\|r"` | 該玩家所有武將技能失效（裝備技除外） |
| `"skillName\|src\|r"` | 該玩家 `skillName` 全部實例失效 |
| `"skillName#2\|src\|r"` | 只失效 `skillName#2`，其他實例正常 |

詳見本文件【第四部分：TriggerV2Skill 使用次數與無效化】。

## 第二部分：被動全域修正技能 (Engine Correct Skills)

### 四種修正類型

| 基類 | 修正目標 | 核心虛擬函式 | 引擎入口 |
|------|---------|-------------|---------|
| `DistanceSkill` | 角色間距離 | `getCorrect(from,to)` / `getFixed(from,to)` | `Engine::correctDistance` |
| `MaxCardsSkill` | 手牌上限 | `getExtra(target)` / `getFixed(target)` | `Engine::correctMaxCards` |
| `TargetModSkill` | 卡牌目標數 / 距離限制 / 使用次數 | `getResidueNum` / `getDistanceLimit` / `getExtraTargetNum` | `Engine::correctCardTarget` |
| `AttackRangeSkill` | 攻擊範圍 | `getExtra(target,include_weapon)` / `getFixed(target,include_weapon)` | `Engine::correctAttackRange` |

### 持有者全場遍歷

引擎對四種修正技能的**非 fixed 分支**，統一遍歷**全場存活角色**（含目標自身），對每一位有同名技能有效實例的玩家疊加貢獻：

```
total = 0
for each skill in getXxxSkills():
    native = skill->getXxx(...)   // 取一次原生基準值
    for each player in getAliveSiblings(true):
        total += sumSkillContribution(player, skill, native)
```

此設計同時處理：
- **自身持有**（如 YingziMaxCards：周瑜自身手牌上限 +1）
- **第三方持有**（如周處技能令全場手牌 -1，實例在周處身上）
- **防禦持有**（如防禦型距離技能在防守方身上）

### 疊加結算規則

```
總貢獻 = Σ 每有效實例的有效貢獻

每實例有效貢獻優先序：
  1. 單實例覆寫     (player setSkillAmountOverride skillName#N)
  2. 全體覆寫       (player setSkillAmountOverride skillName, instanceId=0)
  3. 技能原生回傳值  (skill->getXxx 一次呼叫結果)
```

- `-1` 統一視為 `1000`（用於 TargetModSkill Residue 的「無限」語意）
- **fixed 分支**維持 `max(getFixed(...))` 取最大值，不疊加

### NoInstanceMultiply 屬性

對**自管實例邏輯**的技能（其 `getXxx` 內部已自行依 mark 累加），設定此 property 告知引擎**不進行疊加**：

```cpp
// 在建構子中設定
MyComplexSkill() : DistanceSkill("my_complex") {
    setProperty("NoInstanceMultiply", true);
}
```

`NoInstanceMultiply = true` 時引擎僅**單次套用**原生回傳值，不遍歷持有者實例、不查 modified_amount 覆寫。

### modified_amount 後置覆寫 (Player 層 API)

遊戲中可透過 `Player` 的 API **後置調整**某一技能每實例的貢獻數值，而不修改技能程式碼。

```cpp
// instanceId = 0 為全體覆寫；> 0 為單實例覆寫
void setSkillAmountOverride(const QString &skill_name, int amount, int instanceId = 0);
bool hasSkillAmountOverride(const QString &skill_name, int instanceId = 0) const;
int  getSkillAmountOverride(const QString &skill_name, int instanceId = 0) const;
void removeSkillAmountOverride(const QString &skill_name, int instanceId = 0);
void clearSkillAmountOverrides();
```

**Lua 範例**：
```lua
-- 將某玩家的 mashu 全體改為 -2（每實例 -2）
player:setSkillAmountOverride("mashu", -2, 0)

-- 只改 mashu#1 為 -3
player:setSkillAmountOverride("mashu", -3, 1)
```

### 範例：Mashu 多實例疊加

```cpp
// standard-generals.cpp
class Mashu : public DistanceSkill {
    int getCorrect(const Player *from, const Player *) const {
        if (from->hasSkill(objectName()))
            return -1;   // 每實例 -1 距離
        return 0;
    }
};
```

若玩家有 innate mashu + 2 個 acquired (`mashu#1`/`mashu#2`) = 3 有效實例 → 總貢獻 -3。

## 第三部分：主動觸發技能 (TriggerV2Skill)

### 類層次

```
Skill (QObject)
└── TriggerSkill
    └── TriggerV2Skill
        └── LuaTriggerV2Skill (Lua 綁定)
```

### 觸發流程

```
triggerV2Skills(event, room, target, data)
    │
    ├─ 1. record()      ← 所有 V2 技能（不管是否 triggerable）
    │
    ├─ 2. triggerable() ← 收集可觸發技能（返回 TriggerList）
    │
    ├─ 3. while 循環：
    │     │
    │     ├─ 重新調用所有 v2 技能的 triggerable()
    │     ├─ 解析 multiplier，加入對應數量的 SkillContext
    │     ├─ 注入 trigger_count（已觸發次數）
    │     ├─ 若 skillContexts 為空，break
    │     ├─ askForTriggerOrder ← 玩家選擇
    │     │
    │     └─ 對每個被選中的技能：
    │           │
    │           ├─ cost() → bool
    │           │     └─ 詢問是否發動 + 選目標
    │           │     └─ 返回 false 則跳過
    │           │
    │           ├─ trigger(EventSkillWillInvoke, ctx)
    │           │     └─ 可修改 ctx.is_canceled、ctx.bypass_cost、ctx.multiplier
    │           │
    │           ├─ trigger(EventSkillPay, ctx)
    │           │     └─ 可修改代價相關數值
    │           │
    │           ├─ pay() → bool（若 bypass_cost = true 則跳過）
    │           │     └─ 返回 false 則跳過
    │           │
    │           ├─ trigger(EventSkillTargetConfirming, ctx)
    │           │     └─ 可修改 ctx.updated_targets
    │           │
    │           ├─ trigger(EventSkillInvoking, ctx)
    │           │
    │           ├─ trigger(EventSkillEffect, ctx)
    │           │     └─ 返回 true 則跳過全部效果
    │           │
    │           ├─ effect() → bool（若 EventSkillEffect 返回 true 則跳過）
    │           │     └─ 如 ctx.manual_effect == true，框架不自動遍歷目標
    │           │
    │           ├─ if !manual_effect && targets 不為空：
    │           │     for each target:
    │           │       skillEffect(target) → EventSkillEffectTarget → effectTarget()
    │           │
    │           └─ trigger(EventSkillEffectFinished, ctx)
    │
    └─ 更新 triggerCounts[key]++
```

### TriggerList 與 triggerable 返回格式

```cpp
typedef QMap<ServerPlayer*, QStringList> TriggerList;
```

技能名格式：`source'name*multiplier#instanceId`

| 範例 | skillName | source | multiplier | instanceId |
|------|-----------|--------|------------|------------|
| `baGua` | baGua | - | 1 | 0 |
| `baGua*2` | baGua | - | 2 | 0 |
| `sgs1'baGua*3` | baGua | sgs1 | 3 | 0 |
| `baGua#5` | baGua | - | 1 | 5 |

#### 格式一：單一技能擁有者

```lua
-- 單一技能
return "baGua"

-- 多技能（同一玩家，用 + 分隔）
return "baGua+baGua"

-- 指定玩家（返回兩個值）
return "baGua", ownerPlayer

-- 攜帶 multiplier
return "baGua*3"
```

#### 格式二：多個技能擁有者

```lua
-- 返回兩個字符串，用 | 分隔
return table.concat(trigger_list_skill, "|"),
       table.concat(trigger_list_who, "|")
```

| 欄位 | 格式一 | 格式二 |
|------|--------|--------|
| `ctx.owner` | `target`（事件觸發者） | 技能擁有者 |
| `ctx.invoker` | `target` | `target`（事件觸發者） |

#### 完整範例

```lua
s4_xishe = sgs.CreateTriggerV2Skill{
    name = "s4_xishe",
    events = {sgs.EventPhaseProceeding},
    
    can_trigger = function(skill, event, room, player, data)
        if event == sgs.EventPhaseProceeding then
            if player:getPhase() ~= sgs.Player_Start then return false end
            local trigger_list_skill, trigger_list_who = {}, {}
            for _, owner in sgs.qlist(room:findPlayersBySkillName("s4_xishe")) do
                if owner:objectName() ~= player:objectName() and owner:isAlive()
                   and player:getEquips():length() > 0 and owner:canDiscard(player, "e") then
                    local ctx = sgs.SkillContext()
                    ctx.invoker = owner
                    ctx.owner = owner
                    ctx.instanceID = skill:getInstanceId()
                    if skill:isUsable(ctx) then
                        table.insert(trigger_list_skill, "s4_xishe")
                        table.insert(trigger_list_who, owner:objectName())
                    end
                end
            end
            if #trigger_list_skill > 0 then
                return table.concat(trigger_list_skill, "|"), table.concat(trigger_list_who, "|")
            end
        end
        return false
    end,
    
    on_effect_target = function(skill, event, room, player, ctx, target)
        local slash = sgs.Sanguosha:cloneCard("slash", sgs.Card_NoSuit, 0)
        slash:setSkillName("s4_xishe")
        slash:deleteLater()
        if player:canSlash(target, slash, false) then
            local use = sgs.CardUseStruct()
            use.card = slash
            use.from = player
            use.to:append(target)
            room:useCard(use)
        end
    end
}
```

### 跨角色強制多次觸發 (maxMultipliers)

`EventSkillWillInvoke` 攔截器可以直接修改目標技能的 `ctx.multiplier`，透過 `maxMultipliers` 天花板機制讓技能在本次事件中多觸發。

```lua
-- 技能 A：強制技能 B 多觸發 2 次
force_extra_skill = sgs.CreateTriggerSkill{
    name = "force_extra",
    events = {sgs.EventSkillWillInvoke},
    on_trigger = function(self, event, player, data)
        local ctx = data:toSkillContext()
        if ctx.skill_name == "skill_b" then
            ctx.multiplier = ctx.multiplier + 2
            data:setValue(ctx:toVariant())
        end
    end
}
```

## 第四部分：SkillContext 結構

```cpp
struct SkillContext {
    QString skill_name;            // 發動的技能名稱
    ServerPlayer *invoker;         // 實際發動者（動作執行人）
    ServerPlayer *owner;           // 技能擁有者（技能來源）
    QList<ServerPlayer *> targets;
    QList<ServerPlayer *> updated_targets;  // 目標替換（targetConfirming 用）
    const Card *use_card;
    QVariant *original_data;       // 原始觸發數據（指針）

    int instanceID;                // 實例 ID（區分同名技能）

    ServerPlayer *preferredTarget;
    int preferredTargetSeat;

    bool is_forced;
    bool is_canceled;              // willInvoke 設為 true 可封印技能
    bool bypass_cost;              // willInvoke 設為 true 可跳過代價
    bool manual_effect;            // 設 true 則框架不自動遍歷目標
    TriggerEvent current_event;

    int amount;                    // 技能基礎數值（預設 1）
    int modified_amount;           // 修改後數值（>0 時優先於 amount）
    int trigger_count;             // 已觸發次數
    int multiplier;                // 觸發倍率（maxMultipliers 天花板）

    QString choice;                // askForChoice 結果
    QVariant extra_data;           // 任意額外資料
};
```

### amount 與 modified_amount

TriggerV2Skill 的數值系統與 Engine Correct Skills 的 `sumSkillContribution` 共享同一組概念：

| 系統 | amount 對應 | modified_amount 對應 |
|------|------------|---------------------|
| TriggerV2Skill | `ctx.amount`（由 `base_amount` 填入） | `ctx.modified_amount`（其他技能在 willInvoke 時改） |
| Engine Correct Skills | 技能原生回傳值 | `player->getSkillAmountOverride(name, instanceId)` |

**TriggerV2Skill 數值查詢**：
```cpp
getEffectiveAmount(ctx) 優先序：
1. ctx.modified_amount > 0 → 使用 modified_amount
2. ctx.amount > 0 → 使用 amount
3. 否則 → 使用 skill:getBaseAmount()
```

### Lua 回調參數類型

| 回調 | 第 5 參數類型 | 說明 |
|------|-------------|------|
| `can_trigger` | `data` | `QVariant*`（原始事件數據，用 `data:toDamage()` 等轉換） |
| `on_record` | `ctx` | `SkillContext*`（引用傳遞，直接修改） |
| `on_cost` / `on_pay` / `on_effect` / `on_effect_target` | `ctx` | `SkillContext*`（引用傳遞） |

## 第五部分：技能使用次數限制

適用於 TriggerV2Skill。Engine Correct Skills 無使用次數（全局持續生效）。

### LimitScope 枚舉

| 範圍 | 自動清除時機 |
|------|-------------|
| `Limit_Round` | 輪次結束 |
| `Limit_Turn` | 回合結束 |
| `Limit_Phase` | 指定階段結束 |

### Mark 命名格式

| Scope | Tag Key |
|-------|---------|
| `Limit_Turn` | `Usage_技能名_instanceID-Clear` |
| `Limit_Round` | `Usage_技能名_instanceID_lun` |
| `Limit_Phase` | `Usage_技能名_instanceID-{Phase}Clear` |

### Lua 範例

```lua
local baGua = sgs.CreateTriggerV2Skill{
    name = "baGua",
    events = {sgs.DamageCaused},
    limit_scope = sgs.Skill_Limit_Round,
    max_usage_limit = 2,
    check_custom_usage = function(skill, ctx) return true end,
    can_trigger = function(skill, event, room, player, data)
        if not player:hasSkill(skill:objectName()) then return false end
        local ctx = sgs.SkillContext()
        ctx.invoker = player
        ctx.instanceID = skill:getInstanceId()
        if not skill:isUsable(ctx) then return false end
        return "baGua"
    end,
    on_effect = function(skill, event, room, player, ctx)
        skill:addUsage(ctx)
        return false
    end
}
```

## 第七部分：回調開發規範與常見錯誤

### can_trigger

```lua
can_trigger = function(skill, event, room, player, data)
    -- 1. 檢查玩家是否擁有技能
    if not player:hasSkill(skill:objectName()) then return false end
    
    -- 2. 檢查事件條件
    if event == sgs.Damaged then
        local damage = data:toDamage()
        if not damage or damage.damage < 1 then return false end
        return skill:objectName() .. "*" .. damage.damage
    end
    return false
end
```

### on_cost

```lua
on_cost = function(skill, event, room, player, ctx)
    local choices = {"option1", "option2"}
    local choice = room:askForChoice(player, skill:objectName(), table.concat(choices, "+"))
    ctx.choice = choice
    
    local target = room:askForPlayerChosen(player, room:getOtherPlayers(player), skill:objectName())
    if not target then return false end
    ctx.targets:append(target)
    
    return true
end
```

### on_effect

```lua
on_effect = function(skill, event, room, player, ctx)
    room:sendCompulsoryTriggerLog(player, skill:objectName())
    room:broadcastSkillInvoke(skill:objectName())
    
    -- 取得有效數值
    local amount = skill:getEffectiveAmount(ctx)
    
    -- 若需要手動遍歷目標（包含自己）
    ctx.manual_effect = true
    local targets = sgs.SPlayerList()
    targets:append(player)
    for _, t in sgs.qlist(ctx.targets) do
        targets:append(t)
    end
    room:sortByActionOrder(targets)
    for _, p in sgs.qlist(targets) do
        skill:skillEffect(event, room, player, ctx, p)
    end
    
    return false
end
```

### 常見錯誤

```lua
-- 錯誤：on_cost 將 ctx 當作 data 用
on_cost = function(skill, event, room, player, data)
    local ctx = data:toSkillContext()  -- 錯誤！data 是 SkillContext，不是 QVariant
    return true
end

-- 正確
on_cost = function(skill, event, room, player, ctx)
    return true
end

-- 錯誤：忘記 setValue 寫回原始數據
on_effect = function(skill, event, room, player, ctx)
    local draw = ctx.original_data:toDraw()
    draw.num = draw.num + 1
    -- ctx.original_data:setValue(draw)  -- 漏了
end

-- 正確
on_effect = function(skill, event, room, player, ctx)
    local draw = ctx.original_data:toDraw()
    draw.num = draw.num + 1
    ctx.original_data:setValue(draw)
end
```

## 第八部分：技能無效化 (SkillInvalidity)

適用於 TriggerV2Skill 與 Engine Correct Skills。兩者共用同一套逐實例失效檢查。

### API

```cpp
// Player 層
bool isSkillInvalid(const Skill *skill, int instanceId = 0) const;
bool isSkillInvalid(const QString &skill_name, int instanceId = 0) const;

// Room 層
void addSkillInvalidity(ServerPlayer *target, const QString &skillName,
                        const QString &sourceName, const QString &reason, int instanceId = 0);
void removeSkillInvalidity(ServerPlayer *target, const QString &skillName,
                           const QString &sourceName, const QString &reason, int instanceId = 0);
void clearSkillInvalidityBySource(ServerPlayer *source);
```

### 匹配規則

| Record | 匹配結果 |
|--------|---------|
| `"all"` | 失效所有武將技能（裝備技除外） |
| `"skillName"`（無 #N） | 失效該技能所有實例 |
| `"skillName#2"` | 只失效 `skillName#2` |

### 自動清理

武將死亡時（`BuryVictim`），自動呼叫 `clearSkillInvalidityBySource(player)` 清理該武將造成的所有失效狀態。

### TriggerEvent

| Event | 觸發時機 |
|-------|---------|
| `EventSkillInvalidated` | 技能被無效化時 |
| `EventSkillValidityRestored` | 技能恢復有效時 |

## 第九部分：完整性範例對照

### 場景一：周緒被動手牌上限

需求：周緒在場時，其他玩家手牌上限 -1。周緒可被 acquire 多次。

```cpp
class ZhouxuLimit : public MaxCardsSkill {
public:
    ZhouxuLimit() : MaxCardsSkill("zhouxu_limit") { }

    int getExtra(const Player *target) const override {
        // 引擎 iterates 全場 player，此原生值為每實例的貢獻
        return -1;
    }
};
```

- 周緒有 2 個實例（innate + acquired#1）：其他玩家手牌上限各 -2。
- `NoInstanceMultiply` 不設 → 引擎自動全場查持有者並疊加。

### 場景二：甄姬傾城

需求：傾城發動時，使所有敵方武將技能失效，傾城持有者死亡時自動恢復。

```lua
local Qingcheng = sgs.CreateTriggerV2Skill {
    name = "qingcheng",
    events = {sgs.EventPhaseChanging, sgs.EventSkillInvalidated, sgs.EventSkillValidityRestored},
    can_trigger = function(skill, event, room, player, data)
        if event == sgs.EventPhaseChanging then
            if player:getPhase() == sgs.Player_Start and player:hasSkill("qingcheng") then
                return "qingcheng"
            end
        elseif event == sgs.EventSkillInvalidated then
            -- 返回 true 確認無效化
            return "qingcheng"
        end
        return false
    end,
    on_effect = function(skill, event, room, player, ctx)
        if event == sgs.EventPhaseChanging then
            local source = room:findPlayerBySkillName("qingcheng")
            if source then
                for _, p in sgs.qlist(room:getAlivePlayers()) do
                    if p:getRole() ~= source:getRole() then
                        room:addSkillInvalidity(p, "all", source:objectName(), "qingcheng_active")
                    end
                end
            end
        end
        return false
    end
}
```

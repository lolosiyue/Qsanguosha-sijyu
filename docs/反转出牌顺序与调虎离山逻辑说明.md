# 反转出牌顺序与「调虎离山」逻辑说明

本项目已移植了「反转出牌顺序」功能，并针对「调虎离山」锦囊牌调整了玩家遍历逻辑。

## 1. 核心逻辑变更

### 1.1 玩家指向 (Linked List)
`Room` 类现在维护一个 `bool m_playOrderReversed` 变量。
- 当出牌顺序反转时，遍历 `m_players` 列表并反向连接每个玩家的 `next` 指针。
- 反转操作会自动影响 `getNext()`、`getNextAlive()` 和 `getNextGamePlayer()` 的遍历方向。

### 1.2 调虎离山：不计入座次与距离
「调虎离山」的目标角色会获得 `&heg_lure_tiger-Clear` 标记。
- **`ServerPlayer::getNextAlive(n)`**: 现在会额外跳过拥有该标记的玩家。
    - 影响范围：座次计算、距离计算、围攻 (Encircled) 判断、队列 (Queue) 判断。
- **`ServerPlayer::getNextGamePlayer(n)`**: **新增函数**。仅跳过阵亡玩家，忽略调虎离山标记。
    - 理由：调虎离山的角色虽然不计入座次，但其回合逻辑（如回合开始、回合结束）仍需正常执行。

## 2. 代码实现细节

### ServerPlayer (serverplayer.cpp/h)
- 修正 `getNextAlive` 逻辑：`do next = next->next; while (next->isDead() || (next != this && next->getMark("&heg_lure_tiger-Clear") > 0));`
- 新增 `getNextGamePlayer`：保留原 `getNextAlive` 只跳过死者的逻辑。
- 新增 `getPreviousAlive`：方便 Lua 端或 C++ 端反向搜索存活玩家。

### Room (room.cpp/h)
- `reversePlayOrder()`: 切换反转状态并更新所有玩家的 `next` 指针。
- `isPlayOrderReversed()`: 获取当前是否处于反转状态。

### RoomThread (roomthread.cpp)
- **`actionNormal`**: 翻到下一名玩家时改用 `getNextGamePlayer()`。
    - 确保调虎离山状态下的角色在轮到其回合时，引擎能正确定位到该角色。

## 3. 调试与指令

### 作弊指令 (Cheat Command)
新增了作弊指令 `S_CHEAT_REVERSE_PLAY_ORDER`。可以通过发送对应的作弊协议包来实时切换出牌顺序，方便测试。

### 翻译说明
建议在 Lua 端的翻译文件中增加以下条目：
- `"#ReversePlayOrder"`: "出牌顺序已反转"
- `"#RestorePlayOrder"`: "出牌顺序已恢复"

---
*注意：本功能修改了基础的玩家遍历逻辑，若在开发新技能时遇到座次相关的 Bug，请检查是否使用了正确的遍历函数（`getNextAlive` vs `getNextGamePlayer`）。*

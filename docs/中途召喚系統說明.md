# 中途召喚系統說明

## 概述

「事件疊層最淺層執行」中途召喚系統允許在遊戲進行中，於兩名相鄰玩家之間動態插入新的 AI 玩家。

## 系統架構

```
[Lua Skill]
    → room:requestSummonBetween(before, after, general_name)
    → 參數校驗失敗 → 全部中止（不回滾，直接拋錯）
    → m_pendingSummons.enqueue()
    → 事件迴圈持續執行
    → event_stack.isEmpty() (最淺層)
    → processPendingSummons()
       → insertPlayerMidGame(before, after, general_name)
       → 廣播客戶端
       → doBroadcastNotify(S_COMMAND_ADD_PLAYER_DYNAMIC, info)
       → Client::onPlayerAddedMidGame() → emit player_added()
       → RoomScene::addPlayer() [Dynamic Photo]
       → doBroadcastNotify(S_COMMAND_ARRANGE_SEATS, ...)
       → emit seats_arranged() → arrangeSeats()
```

## 協議指令

| 指令 | 用途 |
|------|------|
| S_COMMAND_ADD_PLAYER_DYNAMIC | 中途召喚新玩家（新增） |
| S_COMMAND_ARRANGE_SEATS | 客戶端重排座次（現有） |

### room.h

新增結構與方法：
```cpp
struct SummonRequest {
    ServerPlayer *before;       // 插入點前的玩家（不可為 nullptr）
    ServerPlayer *after;         // 插入點後的玩家（不可為 nullptr）
    QString general_name;       // 武將名（不可為空）
};

public:
    void requestSummonBetween(ServerPlayer *before, ServerPlayer *after,
                               const QString &general_name);
    void processPendingSummons();
    bool hasPendingSummons() const;

private:
    QList<SummonRequest> m_pendingSummons;
    QList<ServerPlayer*> m_dynamicPlayers;  // 追蹤中途召喚的玩家
    ServerPlayer* insertPlayerMidGame(ServerPlayer *before, ServerPlayer *after,
                                       const QString &general_name);
```

### room.cpp

主要方法實現：
- `hasPendingSummons()` - 檢查是否有待處理的中途召喚請求
- `requestSummonBetween()` - 將召喚請求加入佇列
- `processPendingSummons()` - 處理所有待處理的召喚請求
- `insertPlayerMidGame()` - 執行實際的玩家插入邏輯

`insertPlayerMidGame()` 流程：
1. 參數校驗（before、after 不能為 nullptr，general_name 不能為空）
2. 校驗 before 和 after 是否相鄰
3. 創建新的 ServerPlayer（AI 機器人）
4. 設定玩家屬性（objectName、general、hp、state）
5. 重構環狀鏈結（在 before 和 after 之間插入新玩家）
6. 更新 m_players 和 m_alivePlayers 陣列
7. 廣播 S_COMMAND_ADD_PLAYER_DYNAMIC 通知客戶端
8. 重新分配座次並廣播
9. 廣播 S_COMMAND_ARRANGE_SEATS
10. 註冊觸發技能
11. 發初始手牌（4 張）
12. 將玩家加入 m_dynamicPlayers

新增回調實現：
```cpp
void Client::onPlayerAddedMidGame(const QVariant &player_info)
{
    if (!player_info.canConvert<JsonArray>()) return;
    JsonArray info = player_info.value<JsonArray>();
    if (info.size() < 3) return;

    QString name = info[0].toString();
    QString screen_name = QString::fromUtf8(QByteArray::fromBase64(info[1].toString().toLatin1()));
    QString avatar = info[2].toString();

    if (getPlayer(name)) return;  // 避免重複

    ClientPlayer *player = new ClientPlayer(this);
    player->setObjectName(name);
    player->setScreenName(screen_name);
    player->setProperty("avatar", avatar);
    player->setSeat(m_players.length() + 1);

    m_players << player;
    emit player_added(player);
}
```

### roomscene.cpp

`addPlayer()` 修改為支援動態創建 Photo：
```cpp
void RoomScene::addPlayer(ClientPlayer* player)
{
    // 1. 嘗試填補空位
    for (int i = 0; i < photos.length(); i++) {
        if (photos[i]->getPlayer() == nullptr) {
            photos[i]->setPlayer(player);
            name2photo[player->objectName()] = photos[i];
            if (!Self->hasFlag("marshalling"))
                Sanguosha->playSystemAudioEffect("add-player", false);
            return;
        }
    }

    // 2. 動態創建新 Photo
    Photo* photo = new Photo;
    photo->setZValue(1);
    addItem(photo);

    photo->setPlayer(player);
    photos << photo;
    name2photo[player->objectName()] = photo;

    // 3. 連接信號
    item2player.insert(photo, player);
    connect(photo, SIGNAL(selected_changed()), this, SLOT(updateSelectedTargets()));
    connect(photo, SIGNAL(enable_changed()), this, SLOT(onEnabledChange()));

    if (!Self->hasFlag("marshalling"))
        Sanguosha->playSystemAudioEffect("add-player", false);
}
```

## 斷線重連處理

當玩家在遊戲中斷線並重連時：
1. `reconnect()` 調用 `marshal()` 向重連玩家發送完整的房間狀態
2. `marshal()` 會檢查 `m_dynamicPlayers` 清單
3. 為每個中途召喚的玩家發送 `S_COMMAND_ADD_PLAYER_DYNAMIC`
4. 客戶端收到後創建對應的 Photo，確保視圖同步

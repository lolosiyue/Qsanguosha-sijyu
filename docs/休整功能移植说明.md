# 休整功能移植说明

## 概述
本次将人间版分支中的休整机制移植到当前项目，提供“类死亡但可恢复”的通用能力。

休整状态与真实死亡的区别：
- 休整时角色不应显示黑白化
- 休整时死亡图标应显示 `rest.png`
- 休整可通过接口恢复，并还原休整前技能

---

## 新增/补齐接口

### `Room`（`src/server/room.h` / `src/server/room.cpp`）
- `void restPlayer(ServerPlayer *player, const QString &reason = QString(), bool discard_cards = false);`
- `void directRestPlayer(ServerPlayer *player, const QString &reason = QString(), bool discard_cards = false);`
- `void unrestPlayer(ServerPlayer *player, bool restore_full_hp = true);`
- `bool isRest(ServerPlayer *player) const;`
- `QList<ServerPlayer *> getRestPlayers() const;`

### `Player`（`src/core/player.h` / `src/core/player.cpp`）
- `bool isRest() const;`

实现逻辑：读取动态属性 `RestPlayer`。

---

## 关键行为说明

### `restPlayer`
- 设置 `RestPlayer = true`
- 记录休整原因到 `tag["RestReason"]`
- 记录休整前技能到 `tag["RestPlayerSkills"]`
- 可选弃置手牌（`discard_cards`）
- 广播 `alive` 并复用 `S_COMMAND_KILL_PLAYER` 触发客户端休整表现
- 移除全部技能并重置 AI

### `directRestPlayer`
- 在 `restPlayer` 基础上，将角色从 `m_alivePlayers` 移除并调整座位
- 同步 `updateStateItem()`
- 用在「要立刻把角色移出行動序列」的休整：例如劇本/特殊模式中，角色必須暫時完全離場但之後可恢復。

### `unrestPlayer`
- 设置 `RestPlayer = false`
- 调用 `revivePlayer`
- 从 `RestPlayerSkills` 还原技能
- 按参数决定是否回满体力（`restore_full_hp`）
- 重置 AI

## 建议测试清单

1. 触发 `restPlayer` 后：
   - 角色显示 `rest.png`
   - 角色不变灰
   - 技能被移除
2. 调用 `unrestPlayer` 后：
   - 角色恢复存活表现
   - 技能恢复
   - `restore_full_hp` 参数行为正确
3. 多人局中：
   - `getRestPlayers()` 返回正确
   - `directRestPlayer` 后座位与状态同步正确

## Mowang 技能接入说明

`src/package/mobile.cpp` 中 `Mowang` 已改为使用通用休整接口：

- 在 `BeforeGameOverJudge`：
   - 调用 `room->restPlayer(player, objectName(), true)`
   - 保留 `avatarIcon` 的 mowang 专属变体逻辑
   - 记录 `RestTurn`
   - 设置 `MowangRestActive = true`，用于区分此技能触发的休整

- 在 `EventPhaseStart`：
   - 改为 `Player::RoundStart` 时检查恢复
   - 仅恢复同时满足以下条件的角色：
      - `RestPlayer == true`
      - `MowangRestActive == true`
      - `RestTurn < TurnLengthCount`
   - 调用 `room->unrestPlayer(p, true)` 完成恢复

### 关于 `m_alivePlayers` 的影响

- `restPlayer()` 本身不会移除 `m_alivePlayers`；
- 但 `mowang` 触发点在 `killPlayer()` 的 `BeforeGameOverJudge` 期间，此时角色已经被 `killPlayer()` 预先移出 `m_alivePlayers`。

这在当前机制下是可接受的：
- 休整角色本来就不应参与行动顺序（不在 `m_alivePlayers` 符合预期）；
- 结算判断可通过 `RestPlayer` 属性配合相关逻辑规避误判（例如角色状态展示与恢复流程）；
- 恢复时通过 `unrestPlayer()->revivePlayer()` 将角色重新纳入 `m_alivePlayers`。

若某个玩法要求“休整期间仍参与存活计数但不行动”，应在该玩法中避免使用 `killPlayer` 路径，改为直接在正常存活状态下调用 `restPlayer()`。

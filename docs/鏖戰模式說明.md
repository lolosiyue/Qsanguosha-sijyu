# 鏖戰模式 (Melee Mode)

## 概述

鏖戰模式是一種特殊遊戲規則，在遊戲進入殘局時自動啟動，加快遊戲節奏。

## 啟用方式

在設置頁面勾選「啟用鏖戰模式（殘局桃當殺/閃）」。

## 觸發條件

| 遊戲模式 | 觸發條件 |
|---------|---------|
| 8人或以上身份局 | 場上存活人數 ≤ max(3, 總人數/2) |
| 國戰 | 場上存活人數 ≤ max(3, 總人數/2) |
| 斗地主 | 遊戲時長 ≥ 5 分鐘 下一輪開始時|

不支援 7 人以下身份局與其他特殊模式。

## 效果

進入鏖戰模式後，所有存活玩家獲得以下效果：
1. **桃當【殺】或【閃】**：未轉化的【桃】只能當做【殺】或【閃】使用或打出，不能用來回復體力。
2. **鎖定桃當【殺/閃】**：手牌中的【桃】鎖定視為【殺/閃】(TODO)

## 技術實現

### 核心類

| 類名 | 類型 | 功能 |
|-----|------|-----|
| `MeleeMode` | TriggerSkill | 監聽回合開始、玩家死亡事件，判斷觸發條件 |
| `MeleePeach` | OneCardViewAsSkill | 手牌中的【桃】視為【殺】或【閃】 |
| `MeleeSlashJink` | BasicCard | 殺/閃二合卡，根據當前需求響應 |

### 配置項

- `Config.EnableMeleeMode`：是否啟用鏖戰模式
- 服務器標誌：`E`

### 文件位置

- 頭文件：`src/package/meleemode.h`
- 實現：`src/package/meleemode.cpp`
- 翻譯：`lang/zh_CN/Package/MeleeModePackage.lua`

## 觸發時機

| 事件 | 說明 |
|-----|-----|
| `TurnStart` | 每回合開始檢查是否滿足條件（身份局/國戰） |
| `RoundStart` | 每輪開始檢查是否滿足條件（斗地主） |
| `BuryVictim` | 玩家死亡時立即檢查（身份局/國戰） |
| `EventPhaseStart` | 出牌階段開始時增加出殺次數 |

## 歷史背景

鏖戰模式源自三國殺國戰規則，當場上人數減少且勢力分佈單一時觸發。進入此模式後，桃不可回血，僅能當作殺或閃使用，極大加快遊戲節奏。

本實現目前僅支援鬥地主、8 人或以上身份局與國戰模式。
